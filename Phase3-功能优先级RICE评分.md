# 数字分身社区 · Phase 3 功能优先级评分（RICE）

> 最后更新：2026-02-25

---

## 评分说明

```
RICE = ( Reach × Impact × Confidence ) / Effort

Reach       每月有多少 % 的活跃用户会用到这个功能
Impact      对北极星指标（有效连接数）的影响
            3=决定性  2=显著  1=中等  0.5=轻微
Confidence  对 Reach/Impact 估算的把握程度
Effort      开发工作量（人周，2-3 人小团队）
```

### 关于连接流程（U17-19）的特别说明

RICE 分 20 看起来低，但这是"北极星功能"：
- 低分原因：Reach 只有 30%（不是每个用户每月都会连接）
- 必须建的原因：没有它，产品没有终局价值
- 结论：它需要被前面所有功能"喂饱"才能发挥作用，是漏斗的终点而非起点
